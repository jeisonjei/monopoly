# Простые Python-задачи
 
 ## Как начать работу с репозиторием в GitHub
 
 В проекте Python-код находится в папке `server/`.
 Ниже приведена краткая инструкция по работе с репозиторием и три очень простые задачи по Python.
 
 ### 1. Клонировать репозиторий
 
 Если репозиторий уже размещён в GitHub, его можно скачать на компьютер командой:
 
 ```bash
 git clone <URL_РЕПОЗИТОРИЯ>
 ```
 
 Пример:
 
 ```bash
 git clone https://github.com/jeisonjei/monopoly.git
 ```
 
 После этого открывается папка проекта.
 
 ### 2. Создать отдельную ветку для задачи
 
 Отдельная ветка нужна для того, чтобы не изменять основную ветку проекта.
 
 Пример команды:
 
 ```bash
 git checkout -b child/fix-board-tiles
 ```
 
 Примеры названий веток:
 - `child/add-tests`
 - `child/fix-board-size-constant`
 - `child/change-tile-name`
 
 ### 3. Сделать изменения
 
 После создания ветки вносятся изменения в код и сохраняются файлы.
 
 ### 4. Добавить изменения в git
 
 ```bash
 git add .
 ```
 
 Если требуется добавить только один файл, можно использовать такую команду:
 
 ```bash
 git add server/game/tests.py
 ```
 
 ### 5. Создать коммит
 
 ```bash
 git commit -m "Add simple board tile tests"
 ```
 
 Сообщение коммита рекомендуется делать коротким и понятным.
 
 ### 6. Отправить ветку в GitHub
 
 ```bash
 git push -u origin child/fix-board-tiles
 ```
 
 ### 7. Создать Pull Request в GitHub
 
 В GitHub используется название **Pull Request**.
 
 Порядок действий:
 - открыть репозиторий на GitHub
 - нажать кнопку **Compare & pull request**, если она появилась
 - указать короткий заголовок
 - кратко описать внесённые изменения
 - создать Pull Request
 
 Пример заголовка:
 - `Add simple tests for board tiles`
 
 Пример описания:
 - Added a few beginner-friendly tests for `board_tiles.py`
 - Checked ownable and special tile behavior
 
 ### 8. Попросить взрослого проверить изменения
 
 Проверка со стороны взрослого особенно полезна на начальном этапе обучения.
 Взрослый может проверить код, подсказать ошибки и объяснить команды git.
 
 ## Простые задачи по Python
 
 Ниже оставлены только три очень простые задачи. Они подходят для первого знакомства с кодом проекта.
 
 ### Задача 1. Заменить число `40` на константу
 
 Файл: `server/game/board_tiles.py`
 
 Что требуется сделать:
 - открыть файл `server/game/board_tiles.py`
 - найти строку `normalized_tile_index = tile_index % 40`
 - создать выше константу `BOARD_SIZE = 40`
 - заменить `40` на `BOARD_SIZE`
 - сохранить файл
 
 Что будет изучено:
 - переменные
 - константы
 - функции

 ### Задача 2. Добавить простую функцию `is_special_tile()`

 Файл: `server/game/board_tiles.py`

 Что требуется сделать:

 - найти функцию `is_ownable_tile(tile_index)`

 - внутри функции получить клетку через `get_tile_definition(tile_index)`
 - вернуть `True`, если тип клетки равен `chance` или `community_chest`
 - вернуть `False` в остальных случаях
 - сохранить файл

 Что будет изучено:
 - функции
 - `if`
 - логические условия

 ### Задача 3. Изменить запасное имя клетки

 Файл: `server/game/board_tiles.py`

 Что требуется сделать:
